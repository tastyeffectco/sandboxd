# v0.4.0 isolated-host test runbook (snapshots / restore / fork)

For testing the **`feat/snapshots-fork`** branch end-to-end on a **fresh,
dedicated Ubuntu server** — create → preview → snapshot → restore → fork →
preview. This is v0.4.0 **test** work: do not run it on production or a shared
host, do not merge the branch, and keep v0.3 untouched.

## Server requirements
- Ubuntu **22.04 or 24.04**
- **2 vCPU** min (4 GB RAM preferred), **30 GB+** disk
- A reachable **public IPv4** (sslip.io maps `*.<ip>.sslip.io` → `<ip>`, so no DNS setup)
- Ports **80 and 443 free** (Traefik uses 80; 443 is reserved for an optional TLS follow-up)
- Docker is installed by the script if missing

## Install
```bash
# on the fresh server, as a sudo-capable user:
git clone https://github.com/tastyeffectco/sandboxd.git
cd sandboxd
git checkout feat/snapshots-fork
./scripts/dev/install-v04-ubuntu.sh           # prints a plan, asks to proceed
# non-interactive:  ASSUME_YES=1 ./scripts/dev/install-v04-ubuntu.sh
# override IP/creds: PUBLIC_IP=1.2.3.4 CONSOLE_PASS=mypw ASSUME_YES=1 ./scripts/dev/install-v04-ubuntu.sh
```
It reuses the repo's normal `docker-compose.yml` (traefik + sandboxd + the
`console` profile) and only layers test config (`.env` +
`docker-compose.override.yml`). On success it prints:
- **Console**: `http://console.<ip>.sslip.io/` + login (`demo` / generated password)
- **Preview pattern**: `http://s-<sandbox-id>-<port>.preview.<ip>.sslip.io/`
- **API**: `127.0.0.1:9090` (loopback only — not exposed publicly)

## What the script configures (and why it's safe on a public box)
- **sslip.io** preview domain from the detected public IP — no DNS, no custom domain.
- **Console gated by HTTP basic auth** (Traefik middleware) so the public control
  surface isn't open. Previews stay public (they're meant to be shareable/testable).
- **Edge API router disabled** (`traefik/dynamic/api.yml` → `.v04disabled`); the API
  is reachable only on host loopback. The console proxies `/v1` internally.
- **Dedicated data dir** `/var/lib/sandboxd-v04-test` (workspaces + SQLite + secrets key).
- HTTP on **:80**. TLS is a documented follow-up (below), not required for the test.

## Test checklist
Open `http://console.<ip>.sslip.io/` and log in. The base image ships a working
Vite + React app (`react-standard`) that serves on port 3000, so a created
sandbox previews immediately.

1. **Open console** — log in with the printed `demo` credentials.
2. **Create app** — type a name, *Create app*. (An app with no sandbox is valid.)
3. **Create sandbox** — *Create sandbox* (defaults to `react-standard`, port 3000).
   Wait for the status badge to read **running**.
4. **Run/build a tiny web app on :3000** — the template's dev server already serves
   on 3000. (Optional: submit a task like *"change the heading to Hello v0.4"* and
   watch the logs/build.)
5. **Confirm preview** — the app-detail Preview pane loads the app; or open
   `http://s-<id>-3000.preview.<ip>.sslip.io/` directly. First hit to a stopped
   sandbox shows a brief "warming up" page, then the app.
6. **Stop sandbox** — *Stop*; badge goes **stopped**.
7. **Capture snapshot** — *Snapshot* (requires a stopped source; you just stopped).
   A toast confirms "Snapshot captured." and it appears in the **Snapshots** panel.
8. **Restore from snapshot** — in Snapshots, *Restore* the entry and confirm the
   destructive prompt. This replaces the app's sandbox with a fresh clone.
9. **Confirm restored preview** — start it if stopped; the preview loads from the
   restored workspace.
10. **Fork from snapshot** — *Fork*, give the new app a name.
11. **Confirm independence** — the fork shows as a **separate app** with its own
    sandbox id; changing one app's workspace does not affect the other (the fork's
    `.git` is reset to a clean ownerless repo).
12. **Confirm fork preview** — open the fork's sandbox; its preview URL works at its
    own `s-<fork-sandbox-id>-3000.preview.<ip>.sslip.io`.
13. **Delete test resources** — *Delete sandbox* on each app. (Note: there is no
    delete-app API yet; leftover empty apps are harmless and the teardown wipes
    everything.)
14. **Confirm no orphans / leaks**:
    ```bash
    sudo docker ps -a --filter name=s-          # no stray sandbox containers
    sudo ls /var/lib/sandboxd-v04-test/workspaces   # only expected workspaces
    ```

## TLS (optional follow-up — not required for this test)
The test runs over HTTP on port 80. Real HTTPS needs a wildcard cert, which over
sslip.io is impractical (per-host Let's Encrypt HTTP-01 hits rate limits; wildcard
DNS-01 needs a domain you control — sslip.io isn't). To do TLS later, use a **real
wildcard domain** instead of sslip.io and follow the README "Production / TLS"
section: point `*.preview.yourdomain.com` at the host, enable Traefik's
`websecure` entrypoint + a DNS-01 cert resolver, set `PREVIEW_DOMAIN=yourdomain.com`,
`PREVIEW_ENTRYPOINT=websecure`, `PREVIEW_TLS=true`, and bind port 443.

## Known limitations of this test config
- HTTP only (see TLS above).
- Previews are public (no auth) by design; the console is basic-auth gated.
- No delete-app API — clean up via *Delete sandbox* + teardown.

## Teardown
```bash
cd sandboxd
sudo docker compose --profile console down
sudo rm -rf /var/lib/sandboxd-v04-test
rm -f docker-compose.override.yml traefik/dynamic/v04-auth.yml .env
[ -f traefik/dynamic/api.yml.v04disabled ] && mv traefik/dynamic/api.yml.v04disabled traefik/dynamic/api.yml
```
Then verify nothing lingers: `sudo docker ps -a --filter name=s-` and
`sudo docker images | grep sandboxd`.
